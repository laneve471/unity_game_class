겨울학기 유니티 기초프로그래밍 교육에서 배운 내용을 바탕으로 게임 제작

안녕하세요, 겨울학기 유니티 기초프로그래밍 교육을 수강한 주정은입니다.

매우 간단한 기능 구현 과정에서조차 시행착오를 너무 많이 겪고있어서 개발 진도가 매우 더뎠습니다.
남은 방학 기간동안 처음부터 다시 복습하며 차근차근 유니티에 익숙해지려 합니다.
2주 동안 수고 많이 하셨습니다. 감사합니다.

ㅡㅡㅡ
목표: 1인칭 시점 퍼즐게임 만들기
  여러개의 큐브와 큐브 사이를 이어주는 통로가 존재하는 커다란 맵을 탈출하는 게임
    통로는 벽 한가운데 존재
    몇몇 통로는 기본적으로 지나갈 수 없음
    특정 위치의 버튼을 누르면 맵이 회전하여 새로운 큐브로 넘어갈 수 있음

  맵 속 큐브는 현재 2 x 2 x 2 =8개 존재
    모든 큐브에 아이템이 하나씩 존재
    전부 모아서 조합하는 것이 마지막 퍼즐
    -> 따라서 모든 방을 한 번씩 들러야 함
ㅡㅡㅡ
현재 상태
  기본적인 맵 모델링
  캐릭터 이동 구현
  플레이어 기초 모델링

해결해야 할 과제
  플레이어 모델과 카메라 회전 정도가 다른 문제
    -> 캐릭터를 카메라에 안 보이게 하는 것으로 임시 해결
  플레이어 모델이 맵을 뚫고 떨어짐

  퍼즐 디자인

    풀어야 할 퍼즐 등등

  버튼을 눌러 맵 변경
    맵이 회전하는 기믹
    게이트로 막혀있던 통로가 열리는 기믹

  아이템 수집을 위한 인벤토리 시스템

몇몇 아이디어
  유저들이 맵을 커스텀마이징, 서로의 맵을 플레이
    단순히 만들어놓은 퍼즐을 깨라고 하면 아무도 안 사줄 것 같다.
    여러개의 맵을 만드는 방법도 있겠지만, 간단한 퍼즐이 아니기에 지금으로서는 한계가 있음
    -> 제공되는 맵은 제작자가 보여주는 몇몇 예시일 뿐, 유저들이 더 다채롭고 재미있는 맵을 만들 수도 있지 않을까

  큐브 개수를 계속 늘려 새로운 맵 제공
    일정 레벨까지 큐브 4개, 이후 8개, 9개, 16개 등등 ...
    큐브 개수는 몇 개 정도가 적당한가
    특정 레벨부터 큐브 개수가 바뀐다면 난이도 조절은 어떻게?
